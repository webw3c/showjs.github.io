JS的数据类型

深入讲解 JavaScript 数据类型、继承、闭包等核心基础知识，分析它们的底层原理。这是构建前端工具的基础

数据类型

有8种，7种基础类型，1种引用类型。null、undefined、Number、String、Boolean、Symbol、BigInt基础类型，Object类型又可分为Array(数组对象)、Function(函数对象)、Date(日期对象)、RegExp(正则对象)、Math(数学函数)

数据类型在内存中的存储方式：

  • 基础类型是存放在栈内存中的，被引用或者拷贝时会完全创建一个完全相等的变量
  • 引用类型是破放在堆内存中的，存储的时地址，多个引用指定同一个地址

数据类型检测

方式一：typeof

typeof可以判断基础数据类型（String、Number、Boolean、Symbol、undefined），用判断null时为Object，但null不是一个对象，这个检测结果是JS遗留的历史问题，可以用 === 全等来判断null。

如果用来判断引用类型只能正确的判断 Function，其他的(Array、Date、RegExp、Math)显示的都是Object，不能检测具体的数据类型。

方式二：instanceof

作用

instanceof 运算符用于检测构造函数的 prototype 属性是否出现在某个实例对象的原型链上

语法

object instanceof constructor

**object** - 某个实例对象，**constuctor** - 某个构造函数

instanceof 运算符用来检测 constructor.prototype 是否存在于参数 object 的原型链上。

我们 new 一个对象，那么这个新对象就是它原型链继承上面的对象，通过 instanceof 就能判断这个对象是否是之前那个函数生成的对象，从而判断出这个新对象的数据类型。

方式三：Object.prototype.toString

toString 是 Object 的原型方法，调用该方法可以返回格式为 “[object Xxx]” 的字符串，其中 Xxx 就是对象的类型。对于 Object 对象，直接调用 toString() 就能返回 [object Object]；而对于其他对象，则需要通过 call 来调用，才能返回正确的类型信息。

Object.prototype.toString.call() 可以很好地判断引用类型，甚至可以把 document 和 window 都区分开来。

在写判断条件的时候一定要注意，使用这个方法最后返回统一字符串格式为 “[object Xxx]” ，而这里字符串里面的 “Xxx” ，第一个首字母要大写（注意：使用 typeof 返回的是小写），这里需要多加留意。

Number() 方法的转换规则

  • 如果是布尔值 true 和 false 会分别转换为 1 和 0
  • 如果是数据返回自身
  • 如果是 null 返回 0
  • 如果是 undefined 返回 NaN
  • 如果是字符串，遵循以下规则：
    1. 如果字符串中只包含数字（或者是 0X / 0x 开头的十六进制数字字符串，允许包含正负号），则将其转换为十进制；
    2. 如果字符串中包含有效的浮点格式，将其转换为浮点数值；
    3. 如果是空字符串，将其转换为 0；
    4. 如果不是以上格式的字符串，均返回 NaN；
  • 如果是对象，并且部署了 [Symbol.toPrimitive] ，那么调用此方法，否则调用对象的 valueOf() 方法，然后依据前面的规则转换返回的值；如果转换的结果是 NaN ，则调用对象的 toString() 方法，再次依照前面的顺序转换返回对应的值（Object 转换规则会在下面细讲）。

隐式类型转换

凡是通过

  1. 逻辑运算符 (&&、 ||、 !)
  2. 数学运算符 (+、-、*、/)
  3. 关系操作符 (>、 <、 <= 、>=)
  4. 相等运算符 (==)
  5. if/while 条件的操作

的操作，遇到两个数据类型不一样的情况，都会出现隐式类型转换。

下面着重讲解一下日常用得比较多的“==”和“+”这两个符号的隐式转换规则。

3.2.1 ‘==’ 的隐式类型转换规则【注意是两个等号，不是三个】

  1. 如果类型相同，无须进行类型转换；
  2. 如果其中一个操作值是 null 或者 undefined，那么另一个操作符必须为 null 或者 undefined，才会返回 true，否则都返回 false；// 注意：null 是等于 undefined，看下面的示例代码
  3. 如果其中一个是 Symbol 类型，那么返回 false；因为 Symbol 是唯一值；
  4. 如果一个操作值是 boolean，那么转换成 number；true 转为 1，false 转为 0；例如：false == 0 是 true
  5. 如果一个操作值为 object 且另一方为 string、number 或者 symbol，就会把 object 转为原始类型再进行判断（调用 object 的 valueOf/toString 方法进行转换）。

3.2.2 ‘+’ 的隐式类型转换规则

‘+’ 号操作符，不仅可以用作数字相加，还可以用作字符串拼接。

  1. 仅当 ‘+’ 号两边都是数字时，进行的是加法运算；
  2. 如果两边都是字符串，则直接拼接。

以上两种情况无须进行隐式类型转换。

Object 的转换规则

对象转换的规则，会先调用内置的 [ToPrimitive] 函数，其规则逻辑如下：

  1. 如果部署了 Symbol.toPrimitive 方法，优先调用再返回；
  2. 调用 valueOf()，如果转换为基础类型，则返回；
  3. 调用 toString()，如果转换为基础类型，则返回；

如果都没有返回基础类型，会报错。

总结：
数据类型的基本概念：这是必须掌握的知识点，作为深入理解 JavaScript 的基础。
数据类型的判断方法：typeof 和 instanceof，以及 Object.prototype.toString 的判断数据类型、手写 instanceof 代码片段，这些是日常开发中经常会遇到的，因此你需要好好掌握。
数据类型的转换方式：两种数据类型的转换方式，日常写代码过程中隐式转换需要多留意，如果理解不到位，很容易引起在编码过程中的 bug，得到一些意想不到的结果。
